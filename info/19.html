Билет №19 по информатике

Вопрос 1: Кодирование графической информации. Растровая и векторная графика. Средства и технологии работы с графикой. Создание и редактирование графических информационных объектов средствами графических редакторов, систем презентационной и анимационной графики. Форматы графических файлов. Способы сжатия.

1.1. Кодирование графической информации

Кодирование графической информации — это процесс преобразования визуальной информации в цифровой формат, пригодный для хранения, обработки и передачи компьютером. Основные подходы к кодированию графики — растровый и векторный.

1.2. Растровая графика

  • Принцип: Изображение формируется из множества маленьких точек (пикселей), расположенных по сетке (растру). Каждый пиксель имеет свой цвет.
  • Характеристики:
    • Разрешение: Количество пикселей на единицу длины (например, dpi - точек на дюйм). Чем выше разрешение, тем более детальное и качественное изображение, но и больше объем файла.
    • Глубина цвета (цветовая палитра): Количество бит, используемое для кодирования цвета одного пикселя. Определяет количество возможных цветов. Например, 1 бит – 2 цвета (черно-белый), 8 бит – 256 цветов, 24 бита (True Color) – около 16.7 млн цветов.
  • Преимущества:
    • Реалистичность: отлично подходит для фотографий и сложных изображений с плавными переходами цвета.
    • Простота обработки: легко редактировать отдельные пиксели.
  • Недостатки:
    • При масштабировании (увеличении) теряется качество (пикселизация).
    • Большой объем файлов, особенно при высоком разрешении и большой глубине цвета.
  • Примеры: Фотографии, сканированные изображения.

1.3. Векторная графика

  • Принцип: Изображение формируется из математических описаний геометрических объектов (линий, кривых, многоугольников, окружностей). Эти объекты называются примитивами.
  • Характеристики:
    • Описываются формулами: например, линия задается координатами начальной и конечной точек, толщиной и цветом.
    • Не зависит от разрешения: при масштабировании качество изображения не теряется, так как объекты перерисовываются по новым математическим описаниям.
  • Преимущества:
    • Отличное качество при любом масштабировании.
    • Малый объем файлов, особенно для простых изображений.
    • Идеально для логотипов, чертежей, схем, иллюстраций, шрифтов.
  • Недостатки:
    • Сложность создания реалистичных изображений (фотографий).
    • Более сложная обработка некоторых элементов.
  • Примеры: Логотипы, иконки, схемы, шрифты.

1.4. Средства и технологии работы с графикой

  • Графические редакторы:
    • Растровые: Adobe Photoshop, GIMP, Paint.NET. Используются для редактирования фотографий, ретуши, создания коллажей.
    • Векторные: Adobe Illustrator, CorelDRAW, Inkscape. Используются для создания логотипов, иллюстраций, макетов для печати.
  • Системы презентационной графики: Microsoft PowerPoint, Google Slides, Keynote. Позволяют включать графику в презентации, применять анимацию.
  • Системы анимационной графики: Adobe Animate, Blender, Cinema 4D. Используются для создания 2D/3D анимации, спецэффектов.

1.5. Форматы графических файлов

  • Растровые форматы:
    • JPEG (JPG): Сжатие с потерями. Идеален для фотографий, где небольшие потери качества незаметны, но файл значительно уменьшается.
    • PNG: Сжатие без потерь. Поддерживает прозрачность. Хорош для веб-графики, логотипов, иконок.
    • GIF: Сжатие без потерь. Поддерживает анимацию и прозрачность (один цвет). Ограничен 256 цветами.
    • BMP: Без сжатия или с простым сжатием. Большой объем файла, используется редко.
    • TIFF: Без сжатия или с сжатием без потерь. Часто используется для печати и в полиграфии.
  • Векторные форматы:
    • SVG: Масштабируемая векторная графика. Открытый стандарт, основанный на XML. Идеален для веб.
    • AI: Формат Adobe Illustrator.
    • CDR: Формат CorelDRAW.
    • EPS: Encapsulated PostScript. Используется для обмена графикой между программами.

1.6. Способы сжатия графики

  • Сжатие без потерь (Lossless Compression):
    • Позволяет полностью восстановить исходное изображение после распаковки.
    • Удаляет избыточную информацию, не влияющую на качество.
    • Примеры алгоритмов: RLE (Run-Length Encoding), LZW (алгоритм Лемпеля-Зива-Велча, используется в GIF, TIFF, PNG).
  • Сжатие с потерями (Lossy Compression):
    • Удаляет часть информации, которая считается наименее заметной для человеческого глаза.
    • Восстановление исходного изображения невозможно, но объем файла значительно уменьшается.
    • Примеры алгоритмов: DCT (Discrete Cosine Transform, используется в JPEG).

Вопрос 3: Рассчитать, какое количество страниц текста можно сохранить на дискете в виде архива при заданных размерах страницы, кодовой таблице и коэффициенте сжатия.

3.1. Условия задачи (примерные)

  • Размер дискеты: 1.44 МБ (стандартный размер)
  • Размер страницы текста: 50 строк по 80 символов.
  • Кодовая таблица: KOI8-R (1 символ = 1 байт).
  • Коэффициент сжатия: 3:1 (например, файл уменьшается в 3 раза).

3.2. Расчет

  1. Объем одной страницы текста:
    • Количество символов на странице: 50 строк * 80 символов/строка = 4000 символов.
    • Объем одной страницы в байтах: 4000 символов * 1 байт/символ = 4000 байт.
  2. Объем дискеты в байтах:
    • 1.44 МБ = 1.44 * 1024 КБ = 1.44 * 1024 * 1024 байт ≈ 1 509 949 байт.
  3. Объем текста, который можно хранить на дискете без сжатия:
    • Количество страниц = Объем дискеты / Объем одной страницы
    • Количество страниц = 1 509 949 байт / 4000 байт/страница ≈ 377.49 страниц.
    • Таким образом, без сжатия можно сохранить примерно 377 страниц.
  4. Объем текста, который можно хранить на дискете с учетом сжатия:
    • Поскольку коэффициент сжатия 3:1, это означает, что объем текста после сжатия уменьшается в 3 раза. Значит, на дискете можно хранить в 3 раза больше несжатого текста.
    • Количество страниц с учетом сжатия = Количество страниц без сжатия * Коэффициент сжатия
    • Количество страниц с учетом сжатия = 377.49 * 3 ≈ 1132.47 страниц.

3.3. Ответ

На дискете объемом 1.44 МБ можно сохранить приблизительно 1132 страницы текста при заданных условиях (размер страницы, кодовая таблица и коэффициент сжатия).
