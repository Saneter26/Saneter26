1. Представление текстовой информации в компьютере. Кодировки текстовой информации.

Представление текстовой информации в компьютере:
Вся текстовая информация в компьютере, будь то буквы, цифры, знаки препинания или специальные символы, хранится в закодированном виде. Каждому символу присваивается уникальный двоичный код (последовательность нулей и единиц). Когда вы нажимаете клавишу на клавиатуре, генерируется определенный код, который передается в компьютер. Затем этот код интерпретируется операционной системой и программами, чтобы отобразить соответствующий символ на экране.
Минимальной единицей хранения символа является байт (8 бит), поскольку 8 бит достаточно для кодирования 256 различных символов ($2^8 = 256$).

Кодировки текстовой информации:
Кодировка — это таблица соответствия между символами алфавита и их числовыми (двоичными) кодами. Различные кодировки были разработаны для поддержки разных языков и наборов символов.

  • ASCII (American Standard Code for Information Interchange): * Одна из первых и наиболее распространенных кодировок. * Использует 7 бит для кодирования символов, что позволяет закодировать 128 символов (от 0 до 127). * Включает латинские буквы (заглавные и строчные), цифры, знаки препинания, управляющие символы. * Недостаток: не поддерживает символы национальных алфавитов (например, кириллицу).
  • Расширенные ASCII-кодировки (Code Pages, Кодовые страницы): * Используют все 8 бит байта, что позволяет закодировать 256 символов (от 0 до 255). * Символы с кодами от 0 до 127 (первая половина таблицы) обычно соответствуют ASCII. * Вторая половина таблицы (с 128 по 255) используется для национальных алфавитов (русский, немецкий, французский и т.д.). * **Примеры для кириллицы:**
    • CP866 (DOS-кодировка): Использовалась в MS-DOS.
    • Windows-1251 (CP1251): Стандартная кодировка для русского языка в Windows. Большинство старых текстовых документов и веб-страниц на русском были созданы в этой кодировке.
    • KOI8-R: Ещё одна популярная кодировка для русского языка, часто использовалась в Unix-подобных системах.
    * Недостаток: Несовместимость между различными расширенными ASCII-кодировками. Документ, созданный в Windows-1251, будет некорректно отображаться в KOI8-R без конвертации.
  • Юникод (Unicode): * Современный стандарт кодирования, разработанный для решения проблемы несовместимости между кодировками. * Цель — присвоить уникальный код каждому символу из всех языков мира, а также множеству математических, технических и других символов. * Использует от 1 до 4 байтов для кодирования символов, что позволяет кодировать более миллиона различных символов. * **Основные формы кодирования Юникода:**
    • UTF-8: Наиболее распространенная кодировка в Интернете. Использует от 1 до 4 байтов на символ. Символы ASCII кодируются одним байтом (совместимость с ASCII). Кириллические символы обычно кодируются двумя байтами.
    • UTF-16: Использует 2 или 4 байта на символ. Часто используется в операционных системах (например, Windows).
    • UTF-32: Использует 4 байта на символ для всех символов. Занимает больше места, но обеспечивает фиксированный размер символа.

2. Назначение и основные функции текстовых процессоров.

Текстовый процессор — это прикладная программа, предназначенная для создания, редактирования, форматирования, сохранения и печати текстовых документов. Это основной инструмент для работы с текстами на компьютере.

Назначение текстовых процессоров: * Создание документов различного типа (письма, отчеты, книги, статьи, резюме и т.д.). * Удобное и эффективное редактирование текста. * Придание документу профессионального и эстетически привлекательного вида через форматирование. * Сохранение документов для последующего использования или печати.

Основные функции текстовых процессоров:

  • Ввод и редактирование текста:
    • Набор текста с клавиатуры.
    • Вставка, удаление, копирование, перемещение текста.
    • Поиск и замена текста.
    • Проверка орфографии и грамматики, автокоррекция.
  • Форматирование символов:
    • Изменение шрифта (гарнитура, размер, начертание: полужирный, курсив, подчеркнутый).
    • Цвет текста, эффекты (подстрочный/надстрочный индекс, зачеркнутый).
  • Форматирование абзацев:
    • Выравнивание (по левому краю, по правому, по центру, по ширине).
    • Отступы (первая строка, отступ слева/справа).
    • Междустрочный интервал, интервал до/после абзаца.
    • Маркированные и нумерованные списки.
  • Форматирование страниц:
    • Настройка полей (верхнее, нижнее, левое, правое).
    • Размер и ориентация страницы (книжная/альбомная).
    • Колонтитулы (верхние/нижние), нумерация страниц.
    • Разбиение на разделы и колонки.
  • Вставка объектов:
    • Таблицы, изображения, диаграммы, фигуры, символы, гиперссылки, видео.
    • Объекты WordArt, буквицы.
  • Работа с большими документами:
    • Создание оглавлений, предметных указателей.
    • Ссылки (сноски, перекрестные ссылки).
    • Режим рецензирования (отслеживание изменений, комментарии).
  • Стили:
    • Применение предопределенных или пользовательских стилей для быстрого и единообразного форматирования текста.
  • Сохранение и печать:
    • Сохранение документов в различных форматах (.docx, .doc, .rtf, .txt, .pdf).
    • Предварительный просмотр перед печатью, настройка параметров печати.
  • Слияние документов:
    • Автоматическое создание серии документов (например, писем), комбинируя основной документ с источником данных (списком адресов).
Примеры текстовых процессоров: Microsoft Word, LibreOffice Writer, Google Docs.

3. Практическое задание:

Предполагается, что у вас есть папка "Ваша фамилия" (например, `D:\Моя группа\Ваша фамилия\`).

  1. В папке «Ваша фамилия» создайте текстовый файл «Резюме».
    • Откройте "Проводник" Windows.
    • Перейдите по пути: D:\Моя группа\Ваша фамилия\ (или просто D:\Ваша фамилия\).
    • Щелкните правой кнопкой мыши по пустому месту в этой папке.
    • Выберите "Создать" -> "Текстовый документ".
    • Введите имя файла "Резюме" и нажмите Enter. Убедитесь, что полное имя файла стало "Резюме.txt".
  2. Напишите в файле «Резюме» свои Ф.И.О., дату рождения, адрес.
    • Откройте созданный файл "Резюме.txt" двойным щелчком.
    • Напишите свои данные, например: 
                                  Ф.И.О.: Иванов Иван Иванович
                            Дата рождения: 01.01.2000
                            Адрес: г. Пугачев, ул. Примерная, д. 10, кв. 5
    • Сохраните изменения в файле ("Файл" -> "Сохранить" или Ctrl+S).
    • Закройте текстовый редактор.
  3. Определите, сколько памяти (в битах) занимает растровое изображение размером 32х32 точки, если количество используемых цветов 256.
    • Размер изображения: 32 точек (ширина) * 32 точек (высота) = 1024 пикселя.
    • Количество используемых цветов: 256.
    • Чтобы определить глубину цвета (количество бит на пиксель), используем формулу $N = 2^I$, где $N$ — количество цветов, а $I$ — глубина цвета в битах. $$256 = 2^I$$ $$I = \log_2 256 = 8 \text{ бит/пиксель}$$
    • Общий объем памяти: Количество пикселей * Глубина цвета.
    • Объем памяти = 1024 пикселя * 8 бит/пиксель = 8192 бит.
    • Или в байтах: 8192 бит / 8 бит/Байт = 1024 Байт.
    • Ответ в битах: 8192 бит.
