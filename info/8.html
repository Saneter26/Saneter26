1. Базовые компьютерные программы. Классификация ПО.

Программное обеспечение (ПО) — это совокупность программ, процедур, правил, а также документации, связанных с функционированием системы обработки данных. ПО является неотъемлемой частью компьютерной системы, позволяющей аппаратной части выполнять полезные для пользователя задачи.

Классификация ПО: ПО делится на три основные категории: системное, прикладное и инструментальное.

  1. Системное программное обеспечение (СПО): * Предназначено для управления аппаратными ресурсами компьютера, обеспечения взаимодействия между аппаратным и прикладным ПО, а также для обслуживания самой компьютерной системы. СПО является базой для работы всех остальных программ. * **Примеры:**
    • Операционные системы (ОС): Windows, macOS, Linux, Android, iOS. Обеспечивают базовые функции управления компьютером.
    • Драйверы устройств: Программы, позволяющие операционной системе взаимодействовать с конкретными аппаратными устройствами (принтером, видеокартой, мышью и т.д.).
    • Утилиты (сервисные программы): Программы для обслуживания компьютера: архиваторы, антивирусы, файловые менеджеры, программы для дефрагментации диска, диагностики, очистки системы.
  2. Прикладное программное обеспечение (ППО): * Предназначено для решения конкретных задач пользователя или предметной области. Это программы, с которыми пользователь непосредственно работает. * **Примеры:**
    • Текстовые процессоры: Microsoft Word, LibreOffice Writer.
    • Табличные процессоры: Microsoft Excel, LibreOffice Calc.
    • Программы для презентаций: Microsoft PowerPoint, LibreOffice Impress.
    • Графические редакторы: Adobe Photoshop, GIMP, Paint.
    • Браузеры: Google Chrome, Mozilla Firefox, Microsoft Edge.
    • Программы для работы с базами данных: Microsoft Access, MySQL.
    • Игры.
    • Специализированное ПО: Бухгалтерские программы, САПР (системы автоматизированного проектирования), медицинские информационные системы и т.д.
  3. Инструментальное программное обеспечение (ИПО) / Системы программирования: * Предназначено для разработки, отладки и тестирования новых программ. Эти программы используют программисты. * **Примеры:**
    • Языки программирования и их трансляторы: Компиляторы и интерпретаторы (C++, Python, Java, JavaScript, C#).
    • Интегрированные среды разработки (IDE - Integrated Development Environment): Объединяют текстовый редактор, компилятор/интерпретатор, отладчик и другие инструменты (Visual Studio, IntelliJ IDEA, PyCharm).
    • Библиотеки функций: Наборы готовых программных модулей, которые могут быть использованы разработчиками.

2. Назначение и основные функции архиваторов.

Архиватор — это программа, предназначенная для сжатия данных (создания архивов) и их распаковки. Сжатие данных позволяет уменьшить объем занимаемого ими пространства на диске и ускорить их передачу по сети.

Назначение архиваторов:

  • Экономия дискового пространства: Уменьшение размера файлов для более эффективного хранения.
  • Ускорение передачи данных: Уменьшение размера файлов сокращает время их передачи по сети или копирования на другие носители.
  • Удобство группировки файлов: Объединение нескольких файлов и папок в один архивный файл. Это удобно при передаче набора документов или при создании резервных копий.
  • Защита данных: Многие архиваторы позволяют устанавливать пароль на архив, защищая его содержимое от несанкционированного доступа.
  • Создание самораспаковывающихся архивов (SFX): Архивы, которые содержат встроенный модуль распаковки и могут быть извлечены без установленного архиватора.

Основные функции архиваторов:

  • Сжатие данных (упаковка):
    • Анализ содержимого файлов и применение алгоритмов сжатия для уменьшения их размера.
    • Разные алгоритмы сжатия (например, Zip, Rar, 7z) обеспечивают разную степень сжатия и скорость работы.
  • Распаковка данных (извлечение):
    • Восстановление оригинальных файлов из архива в их исходном размере и состоянии.
  • Добавление/удаление файлов из архива:
    • Возможность добавлять новые файлы в существующий архив или удалять их из него.
  • Обновление файлов в архиве:
    • Замена старых версий файлов в архиве на более новые.
  • Тестирование архива:
    • Проверка целостности архивного файла на предмет повреждений.
  • Создание многотомных архивов:
    • Разбиение большого архива на несколько частей (томов) для удобства записи на носители ограниченного объема (например, флешки, диски) или передачи по частям.
  • Восстановление поврежденных архивов:
    • Некоторые архиваторы имеют функции для частичного восстановления данных из поврежденных архивов.
Примеры популярных архиваторов: WinRAR, 7-Zip, WinZip.

3. Практическое задание:

Для решения этих задач используем те же принципы, что и в Билете №6: 1 символ = 1 байт = 8 бит. Для изображений - количество пикселей умножить на количество бит на пиксель (глубина цвета).

  1. Определите, сколько памяти (в битах) занимает слово «КОМПЬЮТЕР».
    • Слово "КОМПЬЮТЕР" состоит из 9 символов.
    • Предполагаем, что 1 символ = 1 байт.
    • Количество байт: 9 символов * 1 Байт/символ = 9 Байт.
    • Количество бит: 9 Байт * 8 бит/Байт = 72 бита.
  2. Определите, сколько памяти (в битах) занимает растровое изображение размером 1024х768 точек, если глубина цвета 256.
    • Размер изображения: 1024 точек (ширина) * 768 точек (высота) = 786 432 пикселя.
    • Глубина цвета: 256 цветов.
    • Чтобы определить количество бит на пиксель (глубину цвета), используем формулу $N = 2^I$, где $N$ — количество цветов, а $I$ — глубина цвета в битах. $$256 = 2^I$$ $$I = \log_2 256 = 8 \text{ бит/пиксель}$$
    • Общий объем памяти: Количество пикселей * Глубина цвета.
    • Объем памяти = 786 432 пикселей * 8 бит/пиксель = 6 291 456 бит.
    • Или в байтах: 6 291 456 бит / 8 бит/Байт = 786 432 Байт.
    • Ответ в битах: 6 291 456 бит.
