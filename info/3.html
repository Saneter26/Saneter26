1. Понятие операционной системы. Классификация ОС.

Операционная система (ОС) — это комплекс взаимосвязанных системных программ, которые управляют ресурсами компьютера (процессором, памятью, устройствами ввода/вывода) и обеспечивают взаимодействие между аппаратным обеспечением и пользовательскими программами. ОС является основной частью программного обеспечения, без которой компьютер не может функционировать.

Основные функции ОС:

  • Управление файлами и каталогами (создание, удаление, копирование, перемещение, переименование).
  • Управление памятью (распределение памяти между процессами, защита памяти).
  • Управление процессами (планирование выполнения задач, переключение контекста).
  • Управление устройствами ввода/вывода (принтеры, сканеры, клавиатура, мышь и т.д.).
  • Обеспечение пользовательского интерфейса (графический или командной строки).
  • Защита информации и контроль доступа.
  • Обработка прерываний и ошибок.

Классификация ОС:

  1. По типу пользовательского интерфейса:
    • Текстовые (командной строки): Пользователь взаимодействует с ОС, вводя текстовые команды (например, MS-DOS, ранние версии UNIX).
    • Графические (GUI - Graphical User Interface): Пользователь взаимодействует с ОС через графические элементы (окна, иконки, меню) с помощью мыши и клавиатуры (например, Windows, macOS, Ubuntu).
  2. По числу одновременно выполняемых задач (многозадачность):
    • Однозадачные: Могут выполнять только одну программу в один момент времени (например, MS-DOS).
    • Многозадачные: Могут выполнять несколько программ одновременно, переключаясь между ними (например, Windows, macOS, Linux).
  3. По числу одновременно работающих пользователей (многопользовательские):
    • Однопользовательские: Предназначены для работы одного пользователя (например, домашние версии Windows).
    • Многопользовательские: Позволяют нескольким пользователям работать на одном компьютере одновременно, каждый со своим окружением (например, UNIX, Linux, серверные версии Windows).
  4. По типу поддерживаемых процессоров (архитектура):
    • 32-разрядные ОС.
    • 64-разрядные ОС.
  5. По области применения:
    • Настольные (для персональных компьютеров): Windows, macOS, Linux.
    • Серверные: Windows Server, Linux Server, UNIX.
    • Мобильные: Android, iOS.
    • Встраиваемые (для специализированных устройств): RTOS (Real-Time Operating System) для банкоматов, медицинского оборудования и т.п.

2. Назначение и основные функции файловых систем.

Файловая система — это способ организации, хранения и именования данных на носителях информации (жестких дисках, SSD, флеш-накопителях и т.д.). Она определяет, как данные будут записываться, читаться, удаляться и модифицироваться.

Назначение файловых систем: Основное назначение файловой системы — обеспечить удобный и упорядоченный доступ пользователя и программ к информации, хранящейся на диске, абстрагируясь от физического расположения данных. Без файловой системы данные на диске были бы просто хаотичным набором битов.

Основные функции файловых систем:

  • Управление файлами:
    • Создание, удаление, переименование, копирование и перемещение файлов.
    • Отслеживание размера файла и его физического расположения на диске.
    • Управление атрибутами файлов (например, "только для чтения", "скрытый", "системный").
  • Управление каталогами (папками):
    • Создание и удаление каталогов.
    • Организация файлов в иерархическую структуру (дерево каталогов) для более удобного управления.
  • Распределение дискового пространства:
    • Разбиение диска на кластеры (минимальные единицы выделения дискового пространства).
    • Отслеживание свободных и занятых кластеров для эффективного использования дискового пространства.
    • Предотвращение потери данных при сбоях (например, с помощью журналирования).
  • Обеспечение целостности данных:
    • Механизмы для предотвращения повреждения данных и восстановления файловой системы после сбоев (например, FAT, NTFS, ext4 имеют различные уровни защиты).
    • Поддержка метаданных (информация о файле: дата создания, изменения, владелец, права доступа).
  • Поддержка прав доступа и безопасности:
    • Управление разрешениями на чтение, запись и выполнение файлов для различных пользователей и групп (особенно важно в многопользовательских ОС).

Примеры файловых систем: NTFS (Windows), FAT32 (Windows, флеш-накопители), ext2/3/4 (Linux), HFS+/APFS (macOS).

3. Практическое задание:

Для выполнения этого задания потребуется использовать файловый менеджер (например, "Мой компьютер" / "Этот компьютер" в Windows).

  1. Создайте на диске D: папку "Моя группа".
  2. Внутри папки "Моя группа" создайте папку "Моя фамилия" (например, "Петров").
  3. Внутри папки "Моя фамилия" создайте папку "Мои рисунки".
  4. Скопируйте в папку "Мои рисунки" три любых графических файла (например, с расширениями .jpg, .png, .bmp) с любого доступного носителя (другого диска, USB-флешки, интернета).

Структура папок будет выглядеть следующим образом: 

                D:\
                └── Моя группа\
                    └── Моя фамилия\
                        └── Мои рисунки\
                            ├── рисунок1.jpg
                            ├── рисунок2.png
                            └── рисунок3.bmp
(Названия файлов `рисунок1.jpg`, `рисунок2.png`, `рисунок3.bmp` являются примерами. Вы должны использовать реальные графические файлы.)
