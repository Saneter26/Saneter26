1. Файлы и файловая система. Принцип организации файловой системы операционной системы Windows

Файлы и их характеристики:

  • Файл - именованная область данных на носителе информации
  • Атрибуты файла:
    • Имя (до 255 символов в Windows)
    • Расширение (3-4 символа, указывает на тип файла)
    • Размер (в байтах)
    • Дата создания/изменения
    • Атрибуты (только чтение, скрытый, системный)
  • Типы файлов:
    • Исполняемые (.exe, .bat)
    • Документы (.docx, .pdf)
    • Мультимедиа (.mp3, .mp4)
    • Архивы (.zip, .rar)
    • Системные (.dll, .sys)

Файловая система:

Основные функции:

  • Организация хранения файлов
  • Управление свободным пространством
  • Защита и разграничение доступа
  • Обеспечение целостности данных
  • Ведение метаданных

Типы файловых систем:

  • FAT32 (для флешек, ограничение 4ГБ на файл)
  • NTFS (основная для Windows, поддержка прав доступа)
  • exFAT (для внешних накопителей)
  • ReFS (новейшая от Microsoft для серверов)

Принцип организации файловой системы Windows:

  1. Иерархическая структура: Древовидная организация с каталогами и подкаталогами
  2. Логические диски: Разделение на тома (C:, D: и т.д.)
  3. Метаданные:
    • Таблица MFT (Master File Table) в NTFS
    • Записи о местоположении файлов
    • Атрибуты и права доступа
  4. Кластеры: Минимальный размер выделяемого пространства (обычно 4КБ)
  5. Журналирование: Запись изменений в журнал для восстановления
  6. Специальные папки:
    • Пользовательские (Documents, Pictures)
    • Системные (Windows, Program Files)
    • Скрытые (AppData, System Volume Information)

Пример структуры каталогов Windows:

C:\
├── Windows/       (системные файлы)
├── Program Files/ (установленные программы)
├── Users/         (профили пользователей)
│   ├── User1/
│   │   ├── Documents/
│   │   ├── Downloads/
│   │   └── AppData/ (скрытые данные приложений)
├── PerfLogs/      (логи производительности)
└── pagefile.sys   (файл подкачки)

2. Способы хранения информации в компьютере

1. По типу доступа:

  • Оперативная память (RAM):
    • Быстрый произвольный доступ
    • Энергозависимая
    • Используется для временных данных
  • Постоянная память (ROM):
    • Только для чтения (в классическом варианте)
    • Энергонезависимая
    • Хранит BIOS, микропрограммы

2. По технологии хранения:

  • Магнитное хранение:
    • Жесткие диски (HDD)
    • Магнитные ленты
    • Принцип: намагниченные области
  • Полупроводниковое:
    • SSD, флеш-память
    • Ячейки NAND-памяти
    • Быстрее HDD, но ограниченный ресурс записи

3. По назначению:

  • Кэш-память:
    • Сверхбыстрая память малого объема
    • Уровни L1, L2, L3 в процессоре
  • Видеопамять:
    • GDDR в видеокартах
    • Для хранения текстур и буферов

4. По организации данных:

  • Файловое хранение:
    • Файлы в файловой системе
    • Организованы в каталоги
  • Блочное хранение:
    • RAW-устройства (без ФС)
    • Используется СУБД

Иерархия памяти компьютера:

Уровень Тип памяти Объем Время доступа
1 Регистры процессора несколько сотен байт 1 нс
2 Кэш L1 32-64 КБ 2-5 нс
3 Кэш L2 256 КБ - 2 МБ 5-10 нс
4 Кэш L3 4-32 МБ 10-20 нс
5 ОЗУ (RAM) 4-32 ГБ 50-100 нс
6 SSD 128 ГБ - 2 ТБ 50-150 мкс
7 HDD 500 ГБ - 10 ТБ 5-20 мс

3. Создайте текст в текстовом редакторе по образцу

Потребная эффективная мощность N токарного станка

N =
P2 · V 60 · 102 · η

где P2 - сила резания, кг, P2 = 220 кг;

V - скорость резания, м/мин, V = 92 м/мин;

η - КПД станка, η = 0,9;

(1)

Инструкция для создания в текстовом редакторе:

  1. В Microsoft Word перейдите на вкладку "Вставка"
  2. Выберите "Формула" → "Вставить новую формулу"
  3. Для дробей используйте шаблон "Дробь"
  4. Для нижних индексов (P2) используйте сочетание Ctrl+= или кнопку "Индекс"
  5. Греческие буквы (η) вставляются из панели "Символы"
  6. Номер формулы оформляется в рамке с помощью границы абзаца

Пример расчета:

N =
220 · 92 60 · 102 · 0,9
=
20240 5508
≈ 3,67 кВт
